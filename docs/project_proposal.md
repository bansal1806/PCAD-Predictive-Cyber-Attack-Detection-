# Project Proposal
**Title**: Predictive Cyber Attack Detection Using Machine Learning on Network Traffic Data

## 1. Problem Statement
Traditional Intrusion Detection Systems (IDS) are predominantly **reactive**; they alert administrators only after an attack signature is matched or an anomaly threshold is crossed. In high-speed networks, this delay can be catastrophic. Furthermore, rule-based systems struggle with zero-day attacks and require constant manual updates.

**Our proposed solution** is a **Predictive Machine Learning System** that anticipates cyber attacks before they cause significant damage. By analyzing temporal patterns in network traffic flow (using sliding time windows), the model predicts the probability of an attack occurring in the near future (e.g., next 5 minutes). This shifts the cybersecurity paradigm from *detection* to *prevention*.

## 2. Objectives
- To design an end-to-end ML pipeline for processing real-world network traffic data (CIC-IDS 2017).
- To implement authentic feature engineering, specifically **Time-Window Aggregation**, to capture temporal attack behaviors.
- To train and compare supervised learning models (Logistic Regression, Random Forest, XGBoost, LSTM).
- To evaluate performance using strict metrics: F1-Score, Precision, Recall, and ROC-AUC (not just Accuracy).

## 3. Dataset Description
We will strictly use authentic, publicly available research datasets:
1.  **Primary Dataset**: [CIC-IDS 2017](https://www.unb.ca/cic/datasets/ids-2017.html) (Canadian Institute for Cybersecurity).
    -   Contains benign and the most common attacks (Brute Force, DDoS, Botnet, PortScan).
    -   Format: CSV files with 80+ statistical flow features (generated by CICFlowMeter).
2.  **Secondary Dataset (Validation)**: [UNSW-NB15](https://research.unsw.edu.au/projects/unsw-nb15-dataset).
    -   Used to cross-validate the model's robustness on different network architectures.

## 4. Methodology (High-Level)
1.  **Ingestion**: Load strictly authentic CSV data.
2.  **Preprocessing**: Clean NaNs, encode labels, normalize features.
3.  **Transformation**: Convert `(Flow_1, Flow_2, ...)` into `Time_Window_t`.
    -   *Input*: Aggregated stats of window $t$ (Packet Rate, Entropy).
    -   *Target*: Attack label in window $t+1$.
4.  **Modeling**: Train classifiers to map $X_t \rightarrow Y_{t+1}$.
5.  **Evaluation**: Compare models using 10-fold cross-validation or time-series split.

## 5. Expected Outcomes
- A functional ML pipeline that takes raw PCAP/CSV and outputs attack probabilities.
- A comparative study showing that Tree-based (XGBoost) or Sequence-based (LSTM) models outperform baseline Logistic Regression in predictive tasks.
- A set of "lead-time" metrics demonstrating how early an attack can be predicted.
